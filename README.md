# Codex Session Replay

Parses local OpenAI Codex CLI session logs and turns them into a visual timeline — prompts, commentary, tool calls, and file diffs — alongside a live efficiency dashboard, all running entirely on your own machine.

Built for OpenAI Build Week to answer a question every Codex user has: *where exactly did Codex accelerate this build, and by how much?*

## Features

- **Auto-discovered session list** — scans `~/.codex/sessions` recursively, no manual file selection
- **Full timeline view** — user prompts, Codex commentary and final answers, tool calls, and file changes rendered as real GitHub-style diffs (added/removed lines colored), with syntax-highlighted code
- **Efficiency dashboard** — total tokens, cache hit rate, tool call count, and files changed per session, plus a plain-language summary line (e.g. "Built 2 file changes across 8 tool calls in ~3 minutes")
- **Live search** — filters the timeline by keyword across prompts, messages, tool calls, and diffs as you type
- **Session comparison** — select two sessions to compare token usage, cache rate, tool calls, files changed, and duration side by side
- **Export as Markdown** — turns any session's timeline and stats into a shareable `.md` report
- **Collapsible long code blocks** — tool calls and diffs over 15 lines collapse by default with a show more/less toggle
- **Fully local** — no API keys, no external services, no data leaves your machine. Everything reads directly from Codex's own session logs.

## Setup

Requires Node.js 18+.

```
npm install
node server.js
```

Then open `http://localhost:3000`. The sidebar automatically lists every Codex session found under `~/.codex/sessions`. If the list is empty, run any Codex CLI task first to generate a session file.

No sample data is required — the app reads whatever real session logs already exist on the machine it's run on.

## How Codex and GPT-5.6 were used

The majority of this app's functionality was built directly by Codex CLI (GPT-5.6), not written by hand:

- The initial Express backend (recursive session file discovery, JSONL parsing, REST endpoints) and the core frontend timeline renderer were generated by Codex from a single detailed prompt describing the app's purpose and data shape.
- A second Codex session added live search/filtering, built-in syntax highlighting (no external library), and GitHub-style diff coloring — reviewed against the existing code before extending it.
- A third Codex session added the session comparison view and Markdown export feature.
- A fourth session added UI polish: styled buttons, an inline SVG favicon, the plain-language session overview line, and collapsible long code blocks.
- A follow-up session diagnosed and fixed a real state-management bug (the comparison view and single-session view rendering simultaneously instead of one replacing the other) — Codex was given the specific broken behavior, inspected its own prior rendering logic, identified the root cause, and applied a targeted fix.
- A final session reformatted timestamps to readable local time and added session duration as a comparison metric.

Key decisions made along the way: keeping the app fully local with zero external API dependencies (so it works with just Node installed, and never sends session data anywhere), avoiding heavy frontend frameworks in favor of vanilla JS for simplicity and fast judge testing, and building the diff/syntax highlighting from scratch rather than pulling in a large external library.

This project is, fittingly, self-referential: the session where Codex built its own search, highlighting, and diff-coloring features can be viewed inside the tool itself.
